## /novo-gap — Fluxo de início de GAP

1. Ler obrigatoriamente (nesta ordem):
   - `docs/LESSONS_LEARNED.md`
   - `docs/ROADMAP.md` (verificar se GAP já existe)
   - `docs/CODING_STANDARDS.md`
   - Arquivos de contexto específicos do GAP

2. Indicar modelo de IA recomendado:
   ```
   Modelo recomendado: [SWE-1.5 | Claude Sonnet | Claude Opus] — [justificativa em 1 linha]
   ```

3. Apresentar estratégia completa:
   - **Diagnóstico:** o que foi encontrado na análise
   - **Arquivos a criar/modificar:** lista completa com justificativa
   - **Ordem de implementação:** sequência lógica
   - **Decisões técnicas relevantes:** trade-offs, alternativas descartadas
   - **Docs que serão atualizados:** CHANGELOG, ROADMAP e demais conforme REGRA #8
   - **Pergunta:** "APROVADO para iniciar?"

4. Aguardar "APROVADO" explícito do usuário antes de qualquer código.
