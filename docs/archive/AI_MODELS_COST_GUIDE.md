## 🎯 Recomendações por Tipo de Tarefa

### **1. Correções Simples / CRUD / Refatoração Pontual**
- **Menor Custo:** SWE (Free) ou SWE Fast ($)
- **Modelo Ideal:** GPT 5.1 Codex Medium ($)
- **Justificativa:** Tarefas mecânicas não precisam de reasoning profundo

### **2. Lógica de Negócio / Integração / Testes**
- **Menor Custo:** GPT 5.1 Codex Medium ($) ou Gemini 3 Flash High ($)
- **Modelo Ideal:** Claude Sonnet 4.6 Thinking ($$$)
- **Justificativa:** Precisa de raciocínio moderado e consistência

### **3. Arquitetura / Design / Decisões Críticas**
- **Menor Custo:** GPT 5 Medium Thinking ($$) ou Gemini 2.5 Pro ($$)
- **Modelo Ideal:** Claude Opus 4.6 Thinking ($$$)
- **Justificativa:** Requer raciocínio profundo e análise multidimensional

### **4. Análise de Código / Code Review / Segurança**
- **Menor Custo:** GPT 5.1 Codex Low ($$)
- **Modelo Ideal:** Claude Sonnet 4.6 Thinking ($$$)
- **Justificativa:** Precisa de atenção a detalhes e padrões

### **5. Debugging Complexo / Root Cause Analysis**
- **Menor Custo:** GPT 5.2 High Thinking ($$)
- **Modelo Ideal:** Claude Opus 4.6 Thinking ($$$)
- **Justificativa:** Requer raciocínio profundo e investigação

### **6. Documentação / Explicações / Tutoriais**
- **Menor Custo:** SWE Fast ($) ou Kimi K.2.5 (Free)
- **Modelo Ideal:** GPT 4o ($$)
- **Justificativa:** Comunicação clara, não precisa de reasoning profundo

## 💡 Estratégia de Uso Otimizada

**Regra de Ouro:** Sempre começar com o modelo de menor custo adequado, escalar apenas se necessário.

**Fluxo Recomendado:**
1. **Tarefa simples?** → SWE (Free) ou SWE Fast ($)
2. **Precisa de código?** → GPT 5.1 Codex Medium ($)
3. **Precisa de raciocínio?** → GPT 5 Medium Thinking ($$)
4. **Crítico/Complexo?** → Claude Sonnet 4.6 ($$$)
5. **Arquitetura/Design?** → Claude Opus 4.6 ($$$)

**Economia Estimada:** Usar estratégia otimizada pode reduzir custos em até 70% mantendo qualidade.
